## The publish saga (durable — the non-obvious part)

The first publish **failed, and the gate caught it** — no silent half-publish:

- PR #24 merged → `main` (merge commit `4349957`) → publish run `28865742308` **FAILED**:
  `comfy node publish --token` with an empty value → `Option '--token' requires an argument`.
- **Root cause:** `REGISTRY_ACCESS_TOKEN` is an **Environment secret** (environment `env`, no
  protection rules), but the `publish-node` job had **no `environment:` declaration**, so
  `secrets.REGISTRY_ACCESS_TOKEN` resolved empty. (The token was set correctly — just scoped where
  a plain job can't read it.)
- **Fix:** added `environment: env` to `.github/workflows/publish_action.yml`'s job — commit
  `1e08c86` on `main`, workflow-file-only (no `pyproject` change, so it did not auto-retrigger).
- **Re-fired** via `workflow_dispatch` → run **`28866202144` → success.** 0.2.0 published.
- **Verify from a cold seat:** `gh run view 28866202144` (success) · `gh pr view 24` (MERGED) ·
  the registry publisher page shows the node · #25 is CLOSED.
